# subagent-model-tiers

Единый словарь тиров и семейств моделей.
Этот файл описывает не slug-и конкретного harness, а семейства в формате `vendor/model-family`. Сначала выбери тир по типу работы, затем семейство модели внутри этого тира.

## Общие правила

1. Для каждого запуска субагента сначала определи тир по типу работы, затем выбери семейство модели внутри этого тира.
2. Семейства моделей внутри каждого тира перечислены в порядке приоритета. Всегда выбирай первое доступное семейство из списка выбранного тира.
3. Если семейство недоступно в текущем harness, переходи к следующему семейству по списку в рамках того же тира.
4. Не повышай тир, если нет доступного семейства. Если ни одно семейство выбранного тира недоступно, используй либо **auto** режим, либо текущую **inherit** модель.
5. После выбора тира выбери целевую глубину размышлений (`reasoning` / `thinking`) для этого тира.
6. Канонические уровни глубины размышлений в этом файле: `low`, `balanced`, `deep`.
7. Глубина размышлений зависит от модели. Используй только те параметры, варианты или отдельные model ID, которые реально поддерживает выбранная модель в текущем harness.
8. `low` означает минимальную глубину размышлений, если она поддерживается моделью.
9. `balanced` означает `medium`, если такой уровень есть у модели, иначе обычный режим без принудительного `high`.
10. `deep` означает максимальную доступную глубину размышлений, например `high`, thinking variant или другой усиленный режим модели.
11. Если модель не поддерживает настройку глубины размышлений, используй ее обычный режим и не меняй тир из-за отсутствия такой настройки.

## simple

Используй для узких и предсказуемых задач:

- прочитать небольшой набор файлов и вернуть краткое summary;
- извлечь факты, собрать список, переформатировать ответ;
- классифицировать или разложить данные по уже понятной структуре;
- проверить артефакт по явному чеклисту без глубокой неоднозначной аналитики.

Целевая глубина размышлений: `low`.

Подходящие модели:

1. `cursor/composer`
2. `openai/gpt-mini`
3. `openai/gpt-nano`
4. `openai/gpt-luna`
5. `anthropic/claude-haiku`
6. `google/gemini-flash`


## coding

Используй, когда основной объект работы — код:

- реализация задачи;
- правки по ревью;
- ревью diff и текущих правок;
- анализ рисков, регрессий и соответствия критериям приемки;
- объяснение поведения по исходникам и конфигам.

Целевая глубина размышлений: `balanced`.

Подходящие модели:

1. `cursor/composer`
2. `openai/gpt-terra`
3. `openai/codex`
4. `anthropic/claude-sonnet`
5. `google/gemini-pro`

## code-reviewer

Используй, когда нужно независимо проверить чужие изменения по коду:

- ревью diff и текущих правок после реализации;
- поиск рисков, регрессий и расхождений с критериями приемки;
- проверка безопасности, корректности и производительности изменений;
- критика решения без выполнения самой реализации.

Целевая глубина размышлений: `balanced`.

Подходящие модели:

1. `cursor/grok-4.5`
2. `xai/grok`
3. `openai/codex`
4. `openai/gpt-terra`
5. `openai/gpt-sol`
6. `anthropic/claude-sonnet`
7. `anthropic/claude-opus`
8. `anthropic/claude-fable`

## analytics

Используй только для действительно сложной аналитики:

- root cause investigation;
- архитектурный разбор;
- критика плана;
- synthesis по нескольким источникам с неоднозначностью;
- trade-off analysis при конфликтующих ограничениях.

Целевая глубина размышлений: `deep`.

Подходящие модели:

1. `openai/gpt-sol`
2. `openai/gpt`
3. `anthropic/claude-fable`
4. `anthropic/claude-opus`

# AGENTS.md

Репозиторий **ai-bro**: Cursor skills и субагенты для workflow (план, реализация, расследование, рефлексия). Исходники лежат в `.cursor/skills/` и `.cursor/agents/`.

## Skills: изоляция

**Скиллы НЕ МОГУТ ссылаться на файлы других скиллов.**

Каждый скилл — самодостаточный модуль. Все пути, ссылки и упоминания файлов допустимы **только внутри каталога этого скилла** (например `.cursor/skills/bro-do-it/**`).

### Запрещено

- Относительные или абсолютные ссылки на другой скилл: `../bro-give-me-plan/...`, `../../bro-do-it/...`, `.cursor/skills/bro-*` чужого имени.
- Markdown-ссылки, «загрузи», «см.» и аналоги, указывающие на `SKILL.md`, `references/`, `workflows/`, `subagents/` **другого** скилла.
- Упоминания путей вида `bro-<name>/...`, если `<name>` — не текущий скилл (даже без ссылки).

### Разрешено

- Ссылки и include **внутри одного** скилла: `SKILL.md` → `./references/...`, `./workflows/...`, `./subagents/...`.
- Ссылки на `.cursor/agents/`, корень репозитория, `AGENTS.md`, `README.md` — если это не обход изоляции скиллов.

### Если нужен общий текст

1. **Дублируй** нужный фрагмент в каталог текущего скилла (например `references/`), или
2. Вынеси общее в **корень репозитория** (не в чужой `.cursor/skills/<другой-скилл>/`), и ссылайся оттуда.
3. **Не** клади в скилл этапа A правила исполнения этапа B «для удобства» — в A только то, что нужно оркестратору A; исполнение — в скилле B.

### Примеры

```markdown
# ❌ bro-do-it/workflows/plan.md
Правила контрольных точек — [human-checkpoints.md](../../bro-give-me-plan/references/human-checkpoints.md)

# ✅ bro-do-it/workflows/plan.md
Правила контрольных точек — [human-checkpoints.md](./references/human-checkpoints.md)
(файл живёт в bro-do-it/references/)
```

### Эталон для тиров субагентов

Если скилл использует субагентов и ему нужны тиры или семейства моделей:

1. Эталонный файл — `shared-files/subagent-model-tiers.md`.
2. Перед использованием в конкретном скилле **скопируй** его в `./references/subagent-model-tiers.md` внутри этого скилла.
3. В локальной копии допускается только удалить тиры, которые не используются в этом скилле.
4. Текст оставшихся разделов и общих правил сохраняй без изменений.
5. `SKILL.md` и `workflows/*` такого скилла могут ссылаться только на `./references/subagent-model-tiers.md`.
6. **Не** ссылайся из файлов скилла напрямую на `shared-files/subagent-model-tiers.md`.
7. `subagents/*-prompt.md` **не** должны ссылаться на `subagent-model-tiers.md` и **не** должны содержать инструкции по выбору тира или семейства модели. Выбор тира и модели оркестратор выполняет **до** запуска субагента.

### Эталон для правил диалога

Если скилл задаёт вопросы пользователю или предлагает варианты ответов:

1. Эталонный файл — `shared-files/dialog-rules.md`.
2. Перед использованием в конкретном скилле **скопируй** его в `./references/dialog-rules.md` внутри этого скилла.
3. Текст локальной копии сохраняй без изменений.
4. `SKILL.md`, `workflows/*` и другие файлы скилла могут ссылаться только на `./references/dialog-rules.md`.
5. **Не** ссылайся из файлов скилла напрямую на `shared-files/dialog-rules.md`.

При правке любого файла под `.cursor/skills/**` проверяй: нет ли ссылок за пределы каталога текущего скилла.

## Skills: содержимое

Файлы под `.cursor/skills/<name>/` — **конечные инструкции для агента в рантайме**, не документация сопровождения репозитория.

### Запрещено

- Заметки для мейнтейнера: «при правке X обнови Y», «держи в синхроне», «дублировано в …».
- Правила **исполнения** чужого этапа в reference планирующего скилла (и наоборот).
- Один и тот же смысл в нескольких файлах одного скилла с отсылками «см. …» вместо одного канонического `references/*.md` и кратких проверок в `SKILL.md`.

### Разрешено

- Один канонический `references/*.md` на тему.
- В `SKILL.md` (Quality Control, STOP) — **суть проверки своими словами**, без «по файлу X».

## Субагенты (Task)

Промпт в `subagents/*-prompt.md` (блок для `Task`) — **полный контекст**: субагент **не** читает `SKILL.md`, `references/` и файлы скилла.

### Запрещено в теле промпта субагента

- Ссылки на файлы скилла и формулировки «примени правила из …», «см. human-checkpoints».
- Любые «загрузи» / «прочитай» вне текста, который оркестратор передал в `Task`.

### Обязательно

- Все правила, нужные субагенту для ответа, **внутри** передаваемого промпта (допустимо дублировать фрагмент из `references/` при сборке `Task` оркестратором).

## Skills: язык

Скиллы на русском — формулировки на русском. Технические идентификаторы (`[CHECKPOINT]`, `CreatePlan`, пути к файлам) — как есть.

Избегай разговорных англицизмов, если есть нормальный русский термин (не «prose», а «только в тексте плана»; не «implementation-шаг», а «шаг реализации»).

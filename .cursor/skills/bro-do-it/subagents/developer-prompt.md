# Developer prompt template

Use this template when dispatching an developer subagent

## Tool

- Use tool Task (subagent_type=developer)
- Use model 'composer-2.5'
- **NOT** use fast mode

## Prompt template

```
Твоя цель реализовать задачу: <описание задачи>

## Правила работы

1. Держись заявленных рамок, критериев приемки и ограничений.
2. Реализуй только текущую задачу и предпочитай минимальное полноценное изменение, которое закрывает требование.
3. Не добавляй спекулятивные улучшения, побочные рефакторинги и косметические правки, если без них нельзя безопасно завершить задачу.
4. Сохраняй существующее поведение за пределами текущей области изменений.
5. Не перечитывай и не переисследуй уже покрытый контекст только ради перепроверки или "на всякий случай".
6. Если задача неоднозначна, не хватает критичных входных данных, обязательных инструкций или иного контекста для безопасного решения, остановись и явно запроси недостающий контекст у родителя вместо того, чтобы гадать.
7. Если по ходу работы всплыл новый point of interest и без него нельзя безопасно завершить задачу, не делай широкий самостоятельный обход: верни блокер родителю и запроси расширение контекста. Самостоятельный узкий проход допустим только как исключение, если без него нельзя безопасно завершить текущую задачу.

## Формат ответа

**ВСЕГДА** возвращай краткий структурированный отчет:
- Результат
- Что изменено
- Затронутые области
```

# bro-do-it

Реализация поставленной задачи через субагентов.
**ОБЯЗАТЕЛЬНО** используй [субагентов](#субагенты) для реализации и ревью.
**ОБЯЗАТЕЛЬНО** соблюдай [порядок работы](#порядок-работы).

## Порядок работы

### 1. Декомпозиция

Необходимо декомпозировать задачу и сформировать список конкретных шагов для реализации.

Если указан файл плана, загрузи и следуй инструкциям из [plan](./workflows/plan.md)
Иначе загрузи и следуй инструкциям из [fast-fix](./workflows/fast-fix.md)

### 2. Реализация

Реализуй задачу согласно шагам определенных на этапе [декомпозиция](#1-декомпозиция)
**ОБЯЗАТЕЛЬНО** делегируй работу субагенту developer

Проанализируй список полученных шагов:
* **определи** какие шаги можно сгруппировать и выполнить в одном субагенте (допустимо все шаги передать одному субагенту в случае мелкой задачи)
* **определи** порядок выполнения шагов и можно ли часть из них выполнить параллельно (без пересечения изменяемых файлов)

Передавай субагенту только **НЕОБХОДИМЫЙ** контекст для реализации конкретного шага, чтобы он мог сразу перейти к реализации БЕЗ дополнительного анализа контекста и базы знаний.
**НЕ ПЕРЕДАВАЙ** весь контекст разговора при реализации конкретного шага.

После реализации ВСЕХ шагов переходи к [ревью кода](#3-ревью-кода)
Если возникли проблемы при реализации на любом шаге, **останови** процесс, эскалируй и сообщи пользователю о возникших проблемах.
После получения необходимой информации **продолжи** выполнение с места, где остановился.

### 3. Ревью кода

Проведи ревью текущих правок после [реализации](#2-реализация)
**ОБЯЗАТЕЛЬНО** делегируй работу субагенту reviewer

На ревью нужно отправлять **ТОЛЬКО** изменения исходного кода, тестов, конфигов, инфраструктуры.
Изменения документации, readme файлов **НЕ НУЖНО** отправлять на ревью.

Если после ревью есть 'critical' и 'high' замечания, то переходи к [правкам по ревью](#4-правки-по-ревью) **ТОЛЬКО** для этих замечаний.
Иначе переходи к [финализации](#6-финализация).

### 4. Правки по ревью

Внеси правки для устранения 'critical' и 'high' замечаний выявленных на [ревью](#3-ревью-кода)
**ОБЯЗАТЕЛЬНО** делегируй работу субагенту developer

### 5. Повторное ревью

Проведи ревью текущих правок после [правок по ревью](#4-правки-по-ревью)
**ОБЯЗАТЕЛЬНО** делегируй работу субагенту reviewer

Если после повторного ревью опять есть 'critical' и 'high' замечания, **останови** процесс, эскалируй и сообщи пользователю о возникших проблемах.
Иначе переходи к [финализации](#6-финализация).

### 6. Финализация

Опиши что реализовано и на что обратить внимание после реализации.
Если есть замечания после [первичного](#3-ревью-кода) или [повторного](#5-повторное-ревью) ревью, которые не были реализованы - опиши их.
Если задача покрыта тестами, то опиши результаты прохода тестов.
Иначе опиши как проверить реализацию задачи.

## Субагенты

* developer - выполняет реализацию поставленной задачи. **ОБЯЗАТЕЛЬНО** используй промпт [developer-prompt](./subagents/developer-prompt.md)
* reviewer - выполняет ревью выполненной реализации. **ОБЯЗАТЕЛЬНО** используй промпт [reviewer-prompt](./subagents/reviewer-prompt.md)

## Эскалация

При отклонении от процесса, нехватке данных или затруднениях **останови** процесс и сообщи пользователю.

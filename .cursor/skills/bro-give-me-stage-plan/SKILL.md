# Bro Give Me Stage Plan

Твоя задача составить артефакты этапов со детальным списком шагов для реализации плана.
**НЕ ПИШИ** сам код, только составляй план.

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](./references/dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
3. План **ДОЛЖЕН** быть однозначным, без наличия вариаций и вопросов в нём. Все вопросы и вариации **НУЖНО** решать с человеком.
4. План **НЕ ДОЛЖЕН** подменять шаги реализации кодом, diff, патчами или псевдокодом.
5. Если для описания шага требуется фрагмент реализации, шаг **НУЖНО** переписать на более высокий уровень абстракции.
6. Каждый **новый файл** или **новая самостоятельная сущность** — **отдельный шаг**; подключение или использование этой сущности в других местах — **следующий отдельный шаг**. Поле «Зачем это делаем» объясняет роль **конкретного** изменения, а не всей задачи.

## ОГРАНИЧЕНИЯ / STOP

2. **ЗАПРЕЩЕНО** придумывать и додумывать факты или гипотезы. Любая неоднозначность **ДОЛЖНА** быть решена до перехода к составлению плана.

## ПОРЯДОК РАБОТЫ

1. **ПРОЧИТАЙ** файл плана `plan.md`. Если из контекста он не ясен, то **ОСТАНОВИСЬ** и **СПРОСИ** у человека.
2. **СОЗДАЙ** [артефакт этапа](#stagemd) по [шаблону](./references/stage-template.md) и сохрани файл согласно структуре [memory bank](./references/memory-bank.md). **ОБЯЗАТЕЛЬНО** соблюдай [правила составления шагов](./references/step-rules.md).
3.  **ОТПРАВЬ** [артефакт этапа](#stagemd) на ревью субагенту [thinker](#thinker). **ОБЯЗАТЕЛЬНО** используй субагента и формируй для него промпт по шаблону [stage-critic-prompt.md](./subagents/stage-critic-prompt.md)
4.  **ИСПРАВЬ** [артефакт этапа](#stagemd) при наличии замечаний.
5.  **ОСТАНОВИСЬ** и дождись одобрения человека, либо его корректировок.
6.  **ИСПРАВЬ** [артефакт этапа](#stagemd) при наличии замечаний.

## Артефакты

### stage.md

Содержит детальный план реализации конкретного этапа задачи.
**НЕ ДОЛЖЕН** содержать высокоуровневое описание задачи, оно хранятся в [артефакте плана](#planmd).

Шаблон документа: [stage-template.md](./references/stage-template.md)

Формат имени `stage[N]-[slug].md`, где:
* N - номер стадии
* slug - описание этапа, можно указывать слой (frontend, backend), можно указывать модуль и слой. Должно быть понятно по имени файла что именно будет выполняться.

Примеры имени:
- stage1-module2-backend.md
- stage2-module3-rest.md
- stage3-frontend.md

В случае реализации всё в рамках одной стадии, допустимо имя `stage1-all.md`.

## Субагенты

### thinker

**READONLY** субагент для критики и проведения ревью артефактов плана и этапов.
**ДОЛЖЕН** скептически смотреть на план и искать его изъяны.
**ДОЛЖЕН** использовать "умные" модели с высоким уровнем размышления (premium, gpt, opus).
**ОБЯЗАТЕЛЬНО** при вызове субагента в `Task` — использовать промпт [stage-critic-prompt.md](./subagents/stage-critic-prompt.md)

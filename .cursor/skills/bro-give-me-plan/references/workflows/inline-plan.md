# Inline Plan

Описание workflow по формулированию implementation плана без артефактов.
Задача составить план реализации (implementation plan).

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](../dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
3. План **ДОЛЖЕН** быть однозначным, без наличия вариаций и вопросов в нём. Все вопросы и вариации **НУЖНО** решать с человеком.
4. План **НЕ ДОЛЖЕН** подменять шаги реализации кодом, diff, патчами или псевдокодом.
5. При составлении плана **ОБЯЗАТЕЛЬНО** соблюдай структуру шагов и оформляй по [правилам формирования шагов](#правила-формирования-шагов).
6. Если что-то непонятно, **ОБЯЗАТЕЛЬНО** уточни у человека
7. Если что-то нелогично или неправильно, **ОБЯЗАТЕЛЬНО** уточни у человека

## ОГРАНИЧЕНИЯ / STOP

1. **ЗАПРЕЩЕНО** вызывать субагента для ревью, критики или проверки плана.
2. **ЗАПРЕЩЕНО** придумывать и додумывать факты или гипотезы. Любая неоднозначность **ДОЛЖНА** быть решена до перехода к составлению плана.
3. **ЗАПРЕЩЕНО** создавать артефакты плана.

## ПОРЯДОК РАБОТЫ

1. **ПРОЧИТАЙ** [правила формирования шагов](#правила-формирования-шагов).
2. **СОБЕРИ** информацию для составления однозначного плана: код, контекст:
   - Не спрашивай то, что можно установить с высокой уверенностью.
   - Помечай выведенное как выведенное, пока пользователь его не подтвердит.
3. **ЗАПРОСИ** недостающую информацию у человека. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](../dialog-rules.md).
4. **НАПИШИ** в чат шаги плана. Оформляй шаги по [правилам формирования шагов](#правила-формирования-шагов). **НЕ СОЗДАВАЙ** никаких артефактов и файлов.
5. **ОСТАНОВИСЬ** и жди дальнейших указаний от человека. Либо правки плана, либо переход к реализации.

## ПРАВИЛА ФОРМИРОВАНИЯ ШАГОВ

Основной документ с правилами: [steps-rules.md](../step-rules.md).
Для inline плана, **НУЖНО** использовать краткую структуру шагов:
```
## Шаг N

### Где меняем

`<module>/lib/Domain/EntityType.php` (новый enum)

### Что меняем

Добавить enum с **ровно** перечисленными элементами из раздела, фиксирующего конечный результат.
```

Используй только подразделы:
- где меняем
- что меняем

Остальные подразделы не используй.
Мотивация: inline план используется для небольших задач, в которых **ДОЛЖЕН** соблюдаться баланс простоты и полноты информации.

**ЗАПРЕЩЕНО** вставлять шаг типа «Остановка» — для мелких inline-задач он не нужен.

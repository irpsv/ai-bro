# Сохранение плана (CreatePlan)

План **сохраняй только** через tool `CreatePlan`. Frontmatter файла `.plan.md` (`name`, `overview`, `todos`, `isProject`) формирует Cursor из аргументов вызова — **не дублируй** его в markdown.

## Аргументы `CreatePlan`

| Аргумент | Назначение |
|----------|------------|
| `name` | Короткое имя плана |
| `overview` | 1–2 предложения: суть и границы |
| `todos` | Список задач исполнения (см. ниже) |
| `plan` | Только markdown-тело: заголовок и разделы 1–5 |

## `todos` (frontmatter)

**ОБЯЗАТЕЛЬНО** для каждого шага из раздела «4. шаги реализации» — отдельный элемент в аргументе `todos`:

- `id` — стабильный kebab-case (латиница, цифры, дефисы)
- `content` — короткое проверяемое действие
- `status` — `pending` при создании плана

При **обновлении** существующего `.plan.md`: сначала прочитай файл, **не переписывай** `todos` вслепую — сохрани существующие `id`, актуализируй `content`/`status` под новый план.

## `plan` (markdown-тело)

Структура аргумента `plan`:

1. Один заголовок `# …`
2. Разделы **ровно** в порядке из [разделов плана](./plan-format.md#разделы-плана) в `plan-format.md`: `## 1. …` … `## 5. …`
3. Внутри «4. шаги реализации» — подшаги `## Шаг N` по [шаблону шагов](./plan-format.md#4-шаги-реализации) в `plan-format.md`

**ЗАПРЕЩЕНО** в аргументе `plan`:

- `todos:` и любые YAML-списки задач
- блоки `---` … `---` с метаданными или todos
- повтор `name` / `overview` из frontmatter

## Проверка перед вызовом `CreatePlan`

- Каждый `## Шаг N` из тела имеет соответствующий элемент в `todos`
- В тексте `plan` нет строки `todos:` и нет YAML frontmatter

# Plan critic prompt

Используй при вызове thinker для критики и ревью черновика плана.

## Tool

- Use tool Task (subagent_type=thinker)
- Use model 'gpt-5.4-high'
- **NOT** use fast mode

## Prompt template

```
Твоя задача провести ревью плана реализации и покритиковать его с разных сторон.

## READONLY

- **Разрешено**: читать репозиторий; grep/read, чтобы проверить пути к файлам в шагах.
- **Запрещено**: править файлы, коммиты, `CreatePlan`, патчи кода.

## ПРАВИЛА

1. ПРОВЕРЯЙ пути до файлов, они должны быть полными и существующими
2. ПРОВЕРЯЙ согласованность шагов между собой
3. ПРОВЕРЯЙ согласованность шагов с задачей и целевым результатом
4. ПРОВЕРЯЙ адекватную расстановку CHECKPOINT (правильная группировка шагов по слоям реализации)
5. ПРОВЕРЯЙ согласованность критериев приемки с задачей и целевым результатом
6. ПРОВЕРЯЙ согласованность критериев приемки с шагами реализации
7. План НЕ ДОЛЖЕН содержать шаги попутного рефакторинга, ради красоты или "правильного" нейминга
8. План ДОЛЖЕН быть сфокусированным на задаче и НЕ ДОЛЖЕН делать правки на будущее, вероятность которого крайне мала.
9. В каждом шаге плана ДОЛЖНО быть только одно действие (добавление / изменение / удаление); Без смешивания логически разных изменений (создание сущности и её подключение в другом месте — разные шаги); несколько файлов в одном шаге — только при однотипной правке (изменение вызова одного и того же метода в нескольких местах).

## ФОРМАТ ОТВЕТА

Выдай findings, для каждой находки укажи:
- `problem`: что именно не так
- `impact`: чем это грозит
- `evidence`: на чем основан вывод
- `suggestion`: минимальное осмысленное исправление

Если находок нет, напиши явно: `Существенных проблем не найдено`.

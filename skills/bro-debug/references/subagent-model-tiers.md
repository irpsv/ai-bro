# subagent-model-tiers

Единый словарь тиров и семейств моделей.
Этот файл описывает не slug-и конкретного harness, а семейства в формате `vendor/model-family`. Сначала выбери тир по типу работы, затем семейство модели внутри этого тира.

## Общие правила

1. Для каждого запуска субагента сначала определи тир: `simple`, `coding` или `analytics`.
2. Если работаешь в Cursor и нужен тир `simple` или `coding`, **предпочитай** `cursor/composer`.
3. Если выбранное семейство недоступно в текущем harness, возьми совместимое семейство из того же тира.
4. Не повышай тир, если нет доступной модели. Используй либо **auto** режим, либо текущую **inherit** модель.

## analytics

Используй только для действительно сложной аналитики:

- root cause investigation;
- архитектурный разбор;
- критика плана;
- synthesis по нескольким источникам с неоднозначностью;
- trade-off analysis при конфликтующих ограничениях.

Подходящие семейства моделей:

- `openai/gpt`
- `anthropic/claude-opus`
- `google/gemini-pro`

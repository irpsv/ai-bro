# Bro Init Memory Bank

Твоя задача — **первично создать** memory bank проекта: каноническая структура и начальное заполнение контекста.

**НЕ ПОПОЛНЯЙ** уже существующий банк — для этого есть скилл пополнения (`/bro-remember-it`).
**НЕ СОЗДАВАЙ** артефакты планирования в `plans/` — для планов есть скилл планирования (`/bro-give-me-plan`).

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](./references/dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай [структуру memory bank](./references/memory-bank-structure.md) и [правила содержимого](./references/content-rules.md)!
3. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
4. Сначала самостоятельно собери контекст из репозитория.
5. Не спрашивай то, что можно установить с высокой уверенностью.
6. Помечай выведенное как выведенное, пока пользователь его не подтвердит.

## ОГРАНИЧЕНИЯ / STOP

1. **ЗАПРЕЩЕНО** перезаписывать существующий memory bank.
2. **ЗАПРЕЩЕНО** создавать или изменять файлы в `plans/`.
3. **ЗАПРЕЩЕНО** пополнять банк сверх первичного заполнения при init — дальнейшие знания фиксирует `/bro-remember-it`.
4. **ЗАПРЕЩЕНО** придумывать факты о проекте.

## ПОРЯДОК РАБОТЫ

1. **ПРОВЕРЬ**, что `memory-bank/index.md` отсутствует. Если файл есть — **ОСТАНОВИСЬ**, сообщи об этом и **ПРЕДЛОЖИ** `/bro-remember-it` для пополнения.
2. **ПРОЧИТАЙ** [init-memory-bank](./workflows/init-memory-bank.md) и **СОБЛЮДАЙ** его инструкции.
3. После завершения **ПОКАЖИ** человеку итог: что создано, что заполнено, что требует уточнения.

## Quality Control

Перед завершением проверь:

- структура соответствует схеме memory bank;
- `project/overview.md`, `project/tech-stack.md`, `project/architecture.md` содержат осмысленный черновик или подтверждённые факты;
- `plans/` не заполнен планами;
- если банк уже был — он не перезаписан;
- итог заканчивается одним конкретным следующим шагом (обычно `/bro-remember-it` для дальнейших знаний).

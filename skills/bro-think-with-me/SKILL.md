# bro-think-with-me

Используй этот навык, чтобы конкретизировать и структурировать запрос пользователя.
НЕ СОЗДАВАЙ файл плана и другие артефакты.
НЕ МЕНЯЙ ничего в файлах, работай ТОЛЬКО в режиме чтения READONLY.

## Обязательные правила

- Предпочитай краткие конкретные вопросы вместо открытых интервью.
- Если ответ пользователя подсказывает лучшую формулировку — обнови формулировку перед переходом дальше.
- Спрашивай про альтернативы только при реальном компромиссе, архитектурном выборе, риске или необратимом решении.
- Затронутые модули/файлы/слои сначала черновит агент, затем пользователь подтверждает или поправляет.
- Если задача смешивает режимы, выбери доминирующий и отметь второстепенную тему в «Неизвестно / открытые вопросы».

## Порядок работы

**ОБЯЗАТЕЛЬНО** соблюдай и НЕ нарушай порядок пунктов!
**ОБЯЗАТЕЛЬНО** спрашивай по одному вопросу, НЕ спрашивай все разом!

1. СПРОСИ тип обсуждаемой проблемы или запроса. Варианты типов:
   - `фича`
   - `баг`
   - `техдолг`

2. ПРОЧИТАЙ инструкции в зависимости от указанного пользователем типа:
   - `фича` → [workflows/feature.md](workflows/feature.md)
   - `баг` → [workflows/bug.md](workflows/bug.md)
   - `техдолг` → [workflows/refactor-techdebt.md](workflows/refactor-techdebt.md)

3. СОБЕРИ САМОСТОЯТЕЛЬНО необходимую информацию:
   - Не спрашивай то, что можно установить с высокой уверенностью.
   - Помечай выведенное как выведенное, пока пользователь его не подтвердит.

4. ЗАДАЙ вопросы для уточнения неизвестных или неоднозначнных данных:
   - Задавай СТРОГО 1 вопрос за 1 раз!
   - Спрашивай СТРОГО по порядку из выбранной инструкции (шаг 2). Не перескакивай к следующим разделам.
   - Если ответ неполный, оставайся в том же разделе и задай уточняющий вопрос.

5. РАССКАЖИ после каждого ответа:
   - `Подтверждено:` факты, которые пользователь явно подтвердил
   - `Выводы агента:` выводы агента, ещё не подтверждённые пользователем
   - `Неизвестно:` пробелы, предположения для проверки или блокеры

6. НЕ НАВЯЗЫВАЙ ложную определённость. Принимай и помечай:
   - `неизвестно`
   - `нужна проверка`
   - `зависит от X`
   - `предположение`

## Дисциплина по разделам

Для каждого раздела:
1. Внутренне сформулируй цель раздела.
2. Проверь, можно ли заполнить раздел из уже имеющегося контекста.
3. Задай один точечный вопрос только если остаётся пробел в знаниях.
4. Оставайся в разделе, пока не выполнены критерии выхода или пока оставшийся пробел явно не помечен как неизвестное/блокер.
5. Затем переходи к следующему разделу.

## Формат вопросов

Если нужно задать вопросы пользователя **ОБЯЗАТЕЛЬНО** задавай 1 вопрос за 1 раз.
Если есть варианты решений **ПРЕДЛОЖИ** их пользователю.
**ОБЯЗАТЕЛЬНО** нумеруй предлагаемые варианты ответов: 1) 2) 3) ...

Используй tool `AskQuestion` (или аналог) если выполняются ВСЕ условия:
   - вопрос до 500 символов
   - варианты ответов до 200 символов
   - не требуется развернутый ответ
   - не требуется дополнительый контекст

## Формат результата

Итоговый результат **ДОЛЖЕН** содержать ВСЕ разделы из нужного шаблона выбранного на шаге 1-2, если пользователь ЯВНО не указал не включать какой-либо раздел.

# Bro On Board

Твоя задача — познакомить человека с набором скиллов ai-bro: карта команд, типовой флоу и понятный следующий шаг.

**НЕ СОЗДАВАЙ** артефакты планирования в `plans/` — для планов есть скилл планирования (`/bro-give-me-plan`).

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](./references/dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
3. Сначала самостоятельно собери контекст: есть ли уже memory bank, что за проект.
4. Не спрашивай то, что можно установить с высокой уверенностью.
5. Держи онбординг коротким: карта скиллов и следующий шаг, без полного разбора всех workflow.

## ОГРАНИЧЕНИЯ / STOP

1. **ЗАПРЕЩЕНО** создавать или изменять файлы в `plans/`.
2. **ЗАПРЕЩЕНО** придумывать факты о проекте.
3. **ЗАПРЕЩЕНО** уходить в реализацию задачи пользователя во время онбординга.

## ПОРЯДОК РАБОТЫ

1. **ПРОЧИТАЙ** [tour](./workflows/tour.md) и **СОБЛЮДАЙ** его инструкции.
2. После завершения **ПОКАЖИ** человеку итог и один понятный следующий шаг.

## Quality Control

Перед завершением проверь:

- человек понимает, какой скилл когда вызывать;
- статус memory bank ясен;
- если банка нет — человек знает про `/bro-init-memory-bank`;
- если банк есть — человек знает про `/bro-remember-it` для пополнения;
- итог заканчивается одним конкретным следующим шагом.

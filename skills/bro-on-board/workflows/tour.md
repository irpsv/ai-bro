# Tour — знакомство с ai-bro

Короткий онбординг: карта скиллов и предложение инициализировать memory bank.

## Правила workflow

1. Держи ответ компактным: карта → где банк → один следующий шаг.
2. Не читай и не пересказывай внутренние workflow других скиллов целиком.
3. **ЗАПРЕЩЕНО** уходить в реализацию задачи пользователя во время онбординга.

## ПОРЯДОК РАБОТЫ

1. **ПРОЧИТАЙ** [skills-map.md](../skills-map.md).
2. **ПРОВЕРЬ**, есть ли `memory-bank/index.md` в репозитории.
3. **ПОКАЖИ** человеку краткую карту скиллов и типовой флоу из [skills-map.md](../skills-map.md).
4. **СООБЩИ** статус memory bank:
   - если банка **нет** — кратко объясни зачем он нужен и **ПРЕДЛОЖИ** инициализировать;
   - если банк **есть** — скажи, что он уже готов, и для пополнения нужен `/bro-remember-it`.
5. Если банка нет и человек согласен на инициализацию — **ПРОЧИТАЙ** и **ВЫПОЛНИ** [init-memory-bank](./init-memory-bank.md).
6. Если банка нет, но человек отказался — **ЗАВЕРШИ** онбординг с подсказкой, как вернуться к init позже через `/bro-on-board`.
7. **ЗАВЕРШИ** одним конкретным следующим шагом под текущую задачу человека (если она известна) или предложи выбрать этап из карты.

## Критерии завершения

- Человек видит, какие команды за что отвечают.
- Статус memory bank ясен.
- Есть один понятный следующий шаг.

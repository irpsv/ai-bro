# Tour — знакомство с ai-bro

Короткий онбординг: карта скиллов и рекомендация следующего шага.

## Правила workflow

1. Держи ответ компактным: карта → статус банка → один следующий шаг.
2. Не читай и не пересказывай внутренние workflow других скиллов целиком.
3. **ЗАПРЕЩЕНО** создавать или изменять файлы memory bank — для init направляй на `/bro-init-memory-bank`.
4. **ЗАПРЕЩЕНО** уходить в реализацию задачи пользователя во время онбординга.

## ПОРЯДОК РАБОТЫ

1. **ПРОЧИТАЙ** [skills-map.md](../references/skills-map.md).
2. **ПРОВЕРЬ**, есть ли `memory-bank/index.md` в репозитории.
3. **ПОКАЖИ** человеку краткую карту скиллов и типовой флоу из [skills-map.md](../references/skills-map.md).
4. **СООБЩИ** статус memory bank:
   - если банка **нет** — кратко объясни зачем он нужен и **ПРЕДЛОЖИ** следующий шаг: `/bro-init-memory-bank`;
   - если банк **есть** — скажи, что он уже готов, и для пополнения нужен `/bro-remember-it`.
5. **ЗАВЕРШИ** одним конкретным следующим шагом под текущую задачу человека (если она известна) или предложи выбрать этап из карты.

## Критерии завершения

- Человек видит, какие команды за что отвечают.
- Статус memory bank ясен.
- Есть один понятный следующий шаг.

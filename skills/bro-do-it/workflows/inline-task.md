# inline-task

Автономная реализация задачи, переданной прямо из диалога, **без** отдельного артефакта планирования.

## Основные правила

1. Сначала определи пользовательский результат, рамки и критерии приемки из разговора и кода.
2. Не создавай файл плана, план реализации или другой артефакт планирования.
3. Если пользовательский результат понятен, самостоятельно разрешай технические неоднозначности и выбирай консервативный подход по существующему коду.
4. Запрашивай уточнение только при условиях эскалации из принципа автономности основного процесса.
5. Результат workflow — **task payload** для подстановки в `<task payload>` промптов developer и reviewer.

## Краткая сводка пользователю

Перед правками и запуском developer кратко сообщи:

```text
Сделаю так, чтобы:
- <что пользователь сможет сделать или увидеть>
- <какое поведение изменится>

Проверю, что:
- <пользовательский критерий готовности>

Начинаю реализацию.
```

Правила сводки:

- обычно от двух до пяти пунктов суммарно; допускается больше, если иначе потеряются существенные пользовательские сценарии или критерии готовности;
- только пользовательский результат, без названий файлов, библиотек, архитектуры, миграций и внутренних шагов;
- не предлагай варианты и не проси подтверждение;
- для внутренней задачи опиши наблюдаемый эффект: надежность, скорость, отсутствие ошибки или сохранение поведения.

## Task payload

Сформируй payload:

```text
Цель для пользователя: <результат на пользовательском уровне>
Рамки: <что входит и что не входит>
Критерии приемки: <как понять, что результат достигнут>
Ограничения: <безопасность, совместимость и публичные контракты, если есть>
Допущения: <что принято самостоятельно и почему это не меняет пользовательский результат>
Проверка: <уместные тесты, сборки, ручные или статические проверки>
```

Если задача состоит из нескольких частей, перечисли их явно. Не пересказывай весь контекст разговора — передавай только сведения, необходимые для безопасной реализации.

Для reviewer используй тот же payload.

Синхронизация артефактов плана не выполняется.

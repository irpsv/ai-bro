# fast-fix

Реализация задачи **без** артефактов плана.

## Основные правила

1. СНАЧАЛА определи текущую задачу, рамки и критерии приемки из контекста разговора и кода.
2. Если задача, рамки или критерии приемки **неясны**, **остановись** и **запроси уточнение** у пользователя.
3. НЕ создавай отдельный файл или артефакт план.
4. Результат workflow — **task payload** для подстановки в `<задача>` промптов developer и reviewer.

## Подготовка payload

Проанализуй контекст и определи, что нужно реализовать. Если задача состоит из нескольких независимых частей, реши порядок и возможность параллельного выполнения (без пересечения изменяемых файлов).

## Task payload (fast-fix)

Сформируй payload в свободной форме, но обязательно включи:

```
Реализуй задачу: <краткое описание>

Рамки: <что входит и что не входит>
Критерии приемки: <как проверить результат>
Ограничения: <важные ограничения, если есть>
```

Если задача состоит из нескольких частей, перечисли их явно. Не пересказывай весь контекст разговора — только то, что нужно для безопасной реализации.

Для reviewer используй тот же payload.

Синхронизация артефактов плана **не выполняется**.

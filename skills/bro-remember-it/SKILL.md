# Bro Remember It

Твоя задача — инициализировать и пополнять **memory bank**: постоянный слой знаний проекта (контекст, фичи, playbooks).

**НЕ СОЗДАВАЙ** артефакты планирования в `plans/` — для планов используется отдельный скилл планирования.

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](./references/dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай [структуру memory bank](./references/memory-bank-structure.md) и [правила содержимого](./references/content-rules.md)!
3. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
4. Сначала самостоятельно собери контекст из репозитория и существующего банка.
5. Не спрашивай то, что можно установить с высокой уверенностью.
6. Помечай выведенное как выведенное, пока пользователь его не подтвердит.

## ОГРАНИЧЕНИЯ / STOP

1. **ЗАПРЕЩЕНО** нарушать порядок работы и правила workflows.
2. **ЗАПРЕЩЕНО** создавать или изменять файлы в `plans/`.
3. **ЗАПРЕЩЕНО** ссылаться из `features/` на `plans/`.
4. **ЗАПРЕЩЕНО** создавать подкаталоги в `features/` — одна фича, один файл.
5. **ЗАПРЕЩЕНО** придумывать факты о проекте.
6. **ЗАПРЕЩЕНО** читать оба workflow подряд без необходимости. Сначала определи режим, потом читай только нужный workflow.

## ПОРЯДОК РАБОТЫ

1. **ОПРЕДЕЛИ** путь к memory bank (по умолчанию `memory-bank/` в корне репозитория). При необходимости **СПРОСИ** подтверждение.
2. **ОПРЕДЕЛИ** режим по [правилам выбора workflow](#выбор-workflow).
3. **ПРОЧИТАЙ** только выбранный workflow и **СОБЛЮДАЙ** его инструкции.
4. После завершения **ПОКАЖИ** человеку итог: что создано или изменено, что требует уточнения.

## Выбор workflow

**ЗАПРЕЩЕНО** читать оба workflow подряд.

- Если человек явно просит инициализацию или `memory-bank/index.md` **отсутствует** — выбери [init](./workflows/init.md).
- Если `memory-bank/index.md` **существует** или человек просит добавить/обновить знания — выбери [replenish](./workflows/replenish.md).

Если человек просит init, но банк уже есть — **ОСТАНОВИСЬ**, сообщи об этом и **ПРЕДЛОЖИ** workflow пополнения.

## Quality Control

Перед завершением проверь:

- структура соответствует схеме: `index.md`, `project/`, `features/`, `playbooks/`, `plans/`;
- корневой `index.md` ссылается только на `project/*.md`, `features/index.md`, `playbooks/index.md`;
- фичи — плоские файлы `features/<feature-id>.md`, реестр в `features/index.md`;
- playbooks — плоские файлы `playbooks/<slug>.md`, реестр в `playbooks/index.md`;
- нет дублей и выдуманных фактов;
- `plans/` не затронут.

# bro-remember-it

Пополнение **project memory-bank** рекомендациями по итогам выполненных изменений через субагента **thinker**.

**ОБЯЗАТЕЛЬНО** используй [субагента thinker](#субагент-thinker) и промпт [thinker-prompt](./subagents/thinker-prompt.md).
**ОБЯЗАТЕЛЬНО** соблюдай [порядок работы](#порядок-работы).
**ОБЯЗАТЕЛЬНО** проверь результат по [чеклисту](#чеклист).

На выходе — список рекомендаций **КУДА / ЧТО / ПОЧЕМУ**. Без автозаписи в файлы.

## STOP

**ЗАПРЕЩЕНО** править memory-bank, спеки и другие файлы репозитория.
**ЗАПРЕЩЕНО** делать аналитику размещения в memory-bank самому — **ТОЛЬКО** через субагента thinker.
**ЗАПРЕЩЕНО** продолжать без явного пути к memory-bank (см. [определение memory-bank](#1-определение-memory-bank)).
**ЗАПРЕЩЕНО** рекомендовать в memory-bank то, что не является [переиспользуемым продуктовым знанием](#что-кладём-в-memory-bank).

## Что кладём в memory-bank

Только информацию, которую **можно переиспользовать** и которая **полезна в будущих** сессиях/задачах:
- поведение системы и фич, инварианты, контракты, роли, бизнес-ограничения;
- архитектурные решения и их последствия для продукта;
- сценарии, edge cases, интеграции с внешними системами с точки зрения продукта.

**Не** кладём:
- имена переменных,
- детали реализации «как в коде»,
- одноразовые workaround, процессные уроки агента,
- банальности, дубли того, что уже однозначно следует из кода.

## Субагент thinker

* **thinker** — READONLY аналитика: читает memory-bank, изучает изменения, оценивает куда и что положить. **ОБЯЗАТЕЛЬНО** промпт [thinker-prompt](./subagents/thinker-prompt.md).

## Анти-паттерны (нарушение = эскалация)

| Нарушение | Почему плохо |
| --- | --- |
| Оркестратор сам пишет рекомендации КУДА/ЧТО/ПОЧЕМУ | Обход thinker, слабая модель в основном контексте |
| Сразу править файлы memory-bank | Решение о записи — у пользователя |
| Передать thinker весь diff или всю историю чата | Шум и перерасход контекста |
| Рекомендовать запись «про каждый изменённый файл» | Засорение memory-bank |
| Угадать путь memory-bank без подтверждения | Неверные target |

## Порядок работы

### 1. Определение memory-bank

**ОБЯЗАТЕЛЬНО** зафиксируй и **сообщи пользователю**, где находится memory-bank для этого прогона (полный путь от корня репозитория).

Источники (по приоритету):

1. Пользователь явно указал путь или директорию.
2. В репозитории есть очевидный корень (`memory-bank/`, `docs/memory-bank/`, `.memory/`, `docs`, `.ai`, README/index внутри с описанием).
3. Неясно — **остановись** и спроси, **не** вызывай thinker.

Если неясно — **один** вопрос пользователю с **нумерованными** вариантами (подставь найденные в репо кандидаты, иначе типовые):
1) `memory-bank/`
2) `docs/` (или уточни подпапку)
3) другой путь — пользователь укажет
4) memory-bank ещё нет — создадим позже по [bootstrap-schema](./references/bootstrap-schema.md)

Используй `AskQuestion`, если вопрос укладывается в его ограничения; иначе спроси в чате.

После ответа снова **явно** напиши выбранный путь memory-bank и только тогда переходи дальше.

### 2. Сбор входов для thinker

Оркестратор **не** читает memory-bank для анализа размещения — это делает thinker. Оркестратор собирает **пакет входов**:

**A. Memory-bank**

- подтверждённый корень memory-bank (путь);
- при наличии — путь к индексу/README внутри (если пользователь или репо указали).

**B. Что и зачем делалось**

- если есть **файл плана** — передай thinker путь до файла плана. Сам **НЕ ЧИТАЙ** план, **ТОЛЬКО** передай путь до плана;
- если плана **нет** — сам сформируй краткую выжимку **«что и зачем»** по текущей сессии: цель, мотивация, ключевые решения, границы; **не** пересказывай весь чат.

**C. Список ревизий изменений**

Передай субагенту диапазон/список номеров коммитов;
Если нет ни ревизий, ни описания изменений — **остановись** и запроси у пользователя.

### 3. Делегирование thinker

**ОБЯЗАТЕЛЬНО** один вызов субагента thinker по [thinker-prompt](./subagents/thinker-prompt.md).

Передай только пакет из [§2](#2-сбор-входов-для-thinker). **Не** передавай полную историю чата.

Допустим **один** узкий повторный вызов thinker, если не хватает уточнения по 1–2 рекомендациям (без бесконечного цикла).

### 4. Проверка ответа thinker

Убедись, что ответ:

- на русском;
- следует [формату](./references/output-template.md);
- каждая рекомендация содержит **куда / что / почему**;
- нет предложений вне [продуктового знания](#что-кладём-в-memory-bank);
- указаны конфликты/дубликаты с существующими документами (если thinker их нашёл).

Если формат нарушен — один повторный вызов с указанием пробела.

### 5. Финализация (оркестратор → пользователь)

Выдай пользователю:

1. **Где memory-bank** — подтверждённый путь.
2. **Рекомендации** — структурированный список от thinker.
3. **Что обязательно зафиксировать** — пункты с приоритетом `must` (если thinker выделил).

## Эскалация

**Останови** процесс и сообщи пользователю, если:

- не определён memory-bank;
- нет материала об изменениях;
- thinker заблокирован (нет доступа, пустой bank без конвенции — нужен выбор человека);
- запрос сводится к «запиши сам» без подтверждения — напомни про READONLY рекомендаций.

## Чеклист

* Пользователю **явно** назван путь memory-bank (или получено подтверждение после вопроса с вариантами)
* Собран пакет входов: memory-bank root + «что и зачем» (план или выжимка сессии) + сжатые ревизии
* Вызван thinker по [thinker-prompt](./subagents/thinker-prompt.md)
* Ответ thinker в формате КУДА / ЧТО / ПОЧЕМУ
* Рекомендации только про переиспользуемое продуктовое знание
* Файлы memory-bank **не** изменялись автоматически

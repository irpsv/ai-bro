# Правила содержимого memory bank

## Общие правила

1. Пиши кратко и по делу. Каждый файл — самодостаточный фрагмент знаний.
2. Фиксируй только подтверждённые факты: от человека, из репозитория или из явно указанного источника.
3. Помечай выведенное как выведенное, пока человек его не подтвердит.
4. **ЗАПРЕЩЕНО** дублировать одно и то же в разных файлах. Если тема уже есть — обнови существующий файл.
5. **ЗАПРЕЩЕНО** создавать файлы и каталоги вне [структуры memory bank](./memory-bank-structure.md).
6. При упоминании конкретного существующего файла оформляй путь Markdown-ссылкой с относительным путём от текущего документа. Шаблоны, маски и ещё не созданные пути оставляй в формате кода.

## Классификация при пополнении

Перед записью определи тип знания:

| Тип | Куда | Примеры |
|---|---|---|
| Контекст проекта | `project/*.md` | стек, архитектура, глоссарий, сквозное решение |
| Фича | `features/<feature-id>.md` | описание, границы, поведение, решения по фиче |
| Процедура | `playbooks/<slug>.md` | пошаговое типовое действие |
| План работ | `plans/<feature-id>/` | **НЕ** создавай в этом скилле; для планов используй `/bro-give-me-plan` |

## project/decisions.md

Формат записи:

```markdown
## YYYY-MM-DD — Краткое название решения

**Контекст:** ...

**Решение:** ...

**Последствия:** ...
```

Новые записи добавляй в начало файла.

## features/<feature-id>.md

Рекомендуемые разделы внутри одного файла:

- `## Обзор` — зачем фича, границы, статус
- `## Поведение` — требования и сценарии (если есть что фиксировать)
- `## Решения` — решения по фиче с датой (если есть)

При создании новой фичи **ОБЯЗАТЕЛЬНО** добавь строку в `features/index.md`.

## playbooks/<slug>.md

Рекомендуемые разделы:

- `## Когда применять`
- `## Предусловия`
- `## Шаги`
- `## Проверка результата`

При создании нового playbook **ОБЯЗАТЕЛЬНО** добавь строку в `playbooks/index.md`.

## Индексы

`features/index.md` и `playbooks/index.md` — таблицы-реестры. Держи их актуальными при каждом добавлении или переименовании.

Корневой `index.md` обновляй только если появился новый файл в `project/` или изменилась структура разделов.

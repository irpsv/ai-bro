# Init — первичная инициализация memory bank

Первичное создание банка знаний, когда `memory-bank/index.md` ещё не существует.

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](../dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай [структуру memory bank](../memory-bank-structure.md) и [правила содержимого](../content-rules.md)!
3. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
4. Не спрашивай то, что можно установить с высокой уверенностью из репозитория.
5. Помечай выведенное как выведенное, пока пользователь его не подтвердит.

## ОГРАНИЧЕНИЯ / STOP

1. **ЗАПРЕЩЕНО** перезаписывать существующий memory bank. Если банк уже есть — переключись на workflow [replenish](./replenish.md).
2. **ЗАПРЕЩЕНО** создавать артефакты в `plans/` — это зона скилла планирования.
3. **ЗАПРЕЩЕНО** придумывать факты о проекте. Только подтверждённая информация.

## ПОРЯДОК РАБОТЫ

1. **ПРОВЕРЬ**, что `memory-bank/index.md` отсутствует. Если файл есть — **ОСТАНОВИСЬ** и сообщи, что нужен workflow пополнения.
2. **ОПРЕДЕЛИ** путь к банку (по умолчанию `memory-bank/` в корне репозитория). **СПРОСИ** подтверждение у человека.
3. **ОСТАНОВИСЬ** и дождись ответа.
4. **ЗАПУСТИ** скрипт инициализации:
   ```bash
   bash <путь-к-скиллу>/scripts/init-memory-bank.sh <путь-к-банку>
   ```
   Путь к скиллу — каталог, откуда загружен `bro-remember-it` (например `skills/bro-remember-it/` в репозитории ai-bro или аналог после установки).
5. **СОБЕРИ** базовый контекст проекта из репозитория: README, package-файлы, структура каталогов, CI-конфиги.
6. **ЗАПОЛНИ** ключевые файлы в `project/`:
   - `overview.md`
   - `tech-stack.md`
   - `architecture.md`
   Заполняй на основе собранного контекста. Неизвестное оставь с пометкой для уточнения.
7. **СПРОСИ** у человека, что ещё важно зафиксировать сразу: фичи, playbooks, соглашения, глоссарий. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](../dialog-rules.md) — один вопрос за раз.
8. **СОЗДАЙ** запрошенные файлы в `features/` и `playbooks/` по [правилам содержимого](../content-rules.md). Обнови соответствующие `index.md`.
9. **ПРОВЕРЬ** корневой `index.md`: ссылки только на `project/*.md`, `features/index.md`, `playbooks/index.md`.
10. **ПОКАЖИ** человеку итог: что создано, что заполнено, что требует уточнения.

## Критерии завершения

- Структура каталогов соответствует [структуре memory bank](../memory-bank-structure.md).
- `project/overview.md`, `project/tech-stack.md`, `project/architecture.md` содержат осмысленный черновик или подтверждённые факты.
- Человек видит карту банка и понимает, куда добавлять знания дальше.

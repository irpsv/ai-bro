# Replenish — пополнение memory bank

Добавление и обновление знаний в существующем банке.

## ОБЩИЕ ПРАВИЛА

1. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](../dialog-rules.md)!
2. **ОБЯЗАТЕЛЬНО** соблюдай [структуру memory bank](../memory-bank-structure.md) и [правила содержимого](../content-rules.md)!
3. **ОБЯЗАТЕЛЬНО** соблюдай и **НЕ нарушай** [порядок работы](#порядок-работы)!
4. Перед записью **ПРОЧИТАЙ** существующие файлы по теме — не дублируй.
5. Не спрашивай то, что можно установить с высокой уверенностью из репозитория или банка.

## ОГРАНИЧЕНИЯ / STOP

1. **ЗАПРЕЩЕНО** создавать банк с нуля. Если `memory-bank/index.md` отсутствует — **ОСТАНОВИСЬ** и предложи `/bro-init-memory-bank`.
2. **ЗАПРЕЩЕНО** создавать и изменять артефакты в `plans/` — это зона скилла планирования.
3. **ЗАПРЕЩЕНО** ссылаться из `features/` на `plans/`.
4. **ЗАПРЕЩЕНО** создавать подкаталоги в `features/`. Одна фича — один файл `features/<feature-id>.md`.
5. **ЗАПРЕЩЕНО** придумывать факты. Только подтверждённая информация.

## ПОРЯДОК РАБОТЫ

1. **ПРОВЕРЬ**, что `memory-bank/index.md` существует. Если нет — **ОСТАНОВИСЬ** и предложи `/bro-init-memory-bank`.
2. **ПРОЧИТАЙ** корневой `index.md` и релевантные файлы по запросу человека.
3. **УТОЧНИ** у человека, что именно нужно добавить или обновить, если из запроса это неочевидно. **ОБЯЗАТЕЛЬНО** соблюдай [правила диалога](../dialog-rules.md).
4. **КЛАССИФИЦИРУЙ** знание по [правилам содержимого](../content-rules.md):
   - контекст проекта → `project/*.md`
   - фича → `features/<feature-id>.md`
   - процедура → `playbooks/<slug>.md`
   - план работ → **НЕ** этот скилл; предложи скилл планирования
5. **ПРОВЕРЬ**, нет ли уже файла или раздела по этой теме. Если есть — **ОБНОВИ** его, не создавай дубликат.
6. **ЗАПИШИ** изменения. При новой фиче или playbook **ОБНОВИ** соответствующий `index.md`.
7. Если добавлен новый файл в `project/` — **ОБНОВИ** корневой `index.md`.
8. **ПОКАЖИ** человеку: что изменилось, какие файлы затронуты, нужны ли ещё уточнения.

## Обновление vs создание

| Ситуация | Действие |
|---|---|
| Тема уже есть в `project/` | Дополни или исправь существующий файл |
| Фича уже в `features/index.md` | Обнови `features/<feature-id>.md` |
| Новая фича | Создай `features/<feature-id>.md`, добавь строку в `features/index.md` |
| Новая процедура | Создай `playbooks/<slug>.md`, добавь строку в `playbooks/index.md` |
| Сквозное решение | Новая запись в начале `project/decisions.md` |

## Критерии завершения

- Знание записано в правильный раздел без дублей.
- Реестры (`features/index.md`, `playbooks/index.md`) актуальны.
- Человек подтвердил, что зафиксировано то, что нужно.
